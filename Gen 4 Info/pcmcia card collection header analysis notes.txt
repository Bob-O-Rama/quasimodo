00000060  ff ff ff ff Gg Gg Gg Gg  GG GG GG GG GG GG GG GG  |................| <-- Delimiter ?

VV VV = Version?  01 00 for AutoBel Collections, 02 39 for Encode Backup
ID ID = SEEMS like a collection ID number.   For custom collections, the number seems to
        Start with 41 1F ( LSB 1st ) and increase monotinically thru 43 1F
        In AutoBel Cards, the ASCII name includes a collection number, like "1129-1"
LL LL = SEEMS like the length of the collection in bytes, LSB first
        
On On = Offset from start of collection header to nth track, LSB first
Gg Gg = SEEMS like unitiallized memory following FF FF FF FF delimiter 
GG GG = SEEMS like more uninitiallized memory followowing the delimiter
        However in this case, its clearly MIDI data:
        
         90 3f 4c 09 90 40 40 09
         90 44 78 0e 90 3c 78 00
         90 4c 27 00 90 53 2d 0a <-- This is traceable to MIDI data in a track
